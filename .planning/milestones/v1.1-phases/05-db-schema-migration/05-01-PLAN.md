  <action>
Create `supabase/migrations/002_profiles_schema.sql` containing all schema changes in a single file. Follow the structure and conventions from `001_initial_schema.sql` but with the improvements noted in the research (add `to authenticated` role target, wrap `auth.uid()` in subselect for performance).

The file must contain these sections in order:

**Section 1 -- Drop legacy tables:**
- `DROP TABLE IF EXISTS analyses;` (drop first -- it has FK to user_id, same as user_preferences)
- `DROP TABLE IF EXISTS user_preferences;`

**Section 2 -- Enable extensions:**
- `CREATE EXTENSION IF NOT EXISTS moddatetime SCHEMA extensions;`

**Section 3 -- Create profiles table:**
- `id` uuid PK with gen_random_uuid() default
- `user_id` uuid FK to auth.users(id) ON DELETE CASCADE NOT NULL
- `name` text NOT NULL DEFAULT 'Meine Suche' with CHECK (char_length(name) BETWEEN 1 AND 100)
- `preferences` jsonb NOT NULL DEFAULT '{}'
- `is_default` boolean NOT NULL DEFAULT false
- `created_at` timestamptz DEFAULT now() NOT NULL
- `updated_at` timestamptz DEFAULT now() NOT NULL
- Partial unique index: `CREATE UNIQUE INDEX idx_profiles_one_default_per_user ON profiles (user_id) WHERE (is_default = true);`
- Performance index: `CREATE INDEX idx_profiles_user_id ON profiles (user_id);`
- Auto-update trigger: `CREATE TRIGGER handle_profiles_updated_at BEFORE UPDATE ON profiles FOR EACH ROW EXECUTE PROCEDURE moddatetime(updated_at);`

**Section 4 -- Create analyses table (with profile_id FK):**
- Same columns as v1.0 (id, user_id, listing_id, score, breakdown, summary, created_at)
- ADD `profile_id` uuid FK to profiles(id) ON DELETE CASCADE NOT NULL
- Unique constraint: `UNIQUE(user_id, listing_id, profile_id)` (expanded from v1.0's 2-column constraint)
- Performance index: `CREATE INDEX idx_analyses_profile_id ON analyses (profile_id);`

**Section 5 -- RLS policies for profiles:**
- Enable RLS on profiles
- SELECT, INSERT, UPDATE, DELETE policies for authenticated users on own rows
- Use `(SELECT auth.uid()) = user_id` pattern (subselect for query planner performance)
- Add `TO authenticated` role target on all policies

**Section 6 -- RLS policies for analyses:**
- Enable RLS on analyses
- SELECT, INSERT, UPDATE policies for authenticated users on own rows (no DELETE -- cascade handles it)
- Same subselect and role target pattern as profiles

**Section 7 -- set_active_profile() RPC function:**
- `CREATE OR REPLACE FUNCTION set_active_profile(target_profile_id uuid) RETURNS void`
- LANGUAGE plpgsql, SECURITY DEFINER, SET search_path = ''
- Function body: look up profile, verify it exists (raise 'Profile not found'), verify caller owns it via auth.uid() (raise 'Not authorized'), deactivate current default for user, activate target profile
- Use fully qualified `public.profiles` table references (required for SECURITY DEFINER with empty search_path)

Reference the complete migration structure in 05-RESEARCH.md "Code Examples > Complete Migration File Structure" section. Use that as the implementation template, adjusting only if needed.
