Supabase profiles table schema (from Phase 5 migration):
```sql
CREATE TABLE public.profiles (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  name text NOT NULL CHECK (char_length(name) BETWEEN 1 AND 100),
  preferences jsonb NOT NULL DEFAULT '{}',
  is_default boolean NOT NULL DEFAULT false,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
-- Partial unique index: one active profile per user
CREATE UNIQUE INDEX idx_profiles_one_default ON public.profiles (user_id) WHERE is_default = true;
```

  <files>web/src/app/(dashboard)/profiles/actions.ts, web/src/lib/profile-summary.ts</files>
  <action>
Create `web/src/app/(dashboard)/profiles/actions.ts` with 'use server' directive containing these server actions:

1. **createProfile(name: string)**: Insert new profile with `preferencesSchema.parse({})` as defaults. If user has 0 existing profiles, set `is_default: true` on the new row. Otherwise insert with `is_default: false`. Call `revalidatePath('/profiles')` and `revalidatePath('/', 'layout')`. Return the new profile ID.

2. **renameProfile(profileId: string, newName: string)**: Trim name, validate length 1-100 chars (throw if invalid). Update name via `.update({ name })` on the profile row. `revalidatePath('/profiles')`.

3. **deleteProfile(profileId: string)**: Count user's profiles first. If count <= 1, throw "Cannot delete your last profile". Check if the profile being deleted is active (is_default = true). Delete the profile. If it was active, query the oldest remaining profile and call `set_active_profile` RPC on it to auto-activate. `revalidatePath('/profiles')` and `revalidatePath('/', 'layout')`.

4. **duplicateProfile(sourceProfileId: string)**: Read source profile's name and preferences. Insert new row with `name: source.name + " (Copy)"` and cloned preferences. `revalidatePath('/profiles')`. Return the new profile ID.

5. **setActiveProfile(profileId: string)**: Call `supabase.rpc('set_active_profile', { target_profile_id: profileId })`. `revalidatePath('/profiles')` and `revalidatePath('/', 'layout')` to refresh navbar.

6. **saveProfilePreferences(profileId: string, data: Preferences)**: Validate data with `preferencesSchema.parse(data)`. Update the profile's preferences JSONB. `revalidatePath('/profiles')` and `revalidatePath('/', 'layout')`.

All actions must: call `createClient()`, get user via `getUser()`, throw if not authenticated. Import from `@/lib/supabase/server`, `@/lib/schemas/preferences`.

Create `web/src/lib/profile-summary.ts` with:

**generateProfileSummary(prefs: Partial<Preferences>): string** -- Pure deterministic function that reads form values and builds natural-language summary. Include:
- Location and offer type (e.g., "Looking to rent in Zurich")
- Property type if not ANY

- Rooms range with dealbreaker flag
- Living space range with dealbreaker flag
- Floor preference if not "any"
- Features list (first 3 + count)
- Critical importance categories
- Return "No preferences set yet." if empty

Import `type Preferences` from `@/lib/schemas/preferences`. Export the function.
