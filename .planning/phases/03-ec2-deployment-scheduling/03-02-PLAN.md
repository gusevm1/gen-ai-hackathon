    Script structure (all operations idempotent — check before create):

    ```bash
    #!/usr/bin/env bash
    set -euo pipefail

    REGION="eu-central-1"
    INSTANCE_TYPE="t3.medium"
    KEY_NAME="swiss-scraper-key"
    SG_NAME="swiss-scraper-sg"
    VOLUME_SIZE=20
    S3_BUCKET="swiss-property-scraper-data"
    IAM_ROLE="swiss-scraper-ec2-role"
    INSTANCE_PROFILE="swiss-scraper-s3-profile"
    ```

    Steps (each with idempotency check):
    a. **Get latest AL2023 AMI** via SSM Parameter Store:
       `aws ssm get-parameter --name "/aws/service/ami-amazon-linux-latest/al2023-ami-kernel-default-x86_64"`

    b. **Create key pair** (skip if exists):
       Check with `aws ec2 describe-key-pairs --key-names`. If not found, create and save .pem file with chmod 400.

    c. **Create security group** (skip if exists):
       - Inbound: TCP 22 from operator IP (auto-detected via `curl -s ifconfig.me`)
       - Outbound: all (default)
       Check with `aws ec2 describe-security-groups --filters "Name=group-name,Values=$SG_NAME"`.

    d. **Create IAM role + instance profile** (skip if exists):
       - Trust policy: EC2 can assume the role
       - Attach inline policy: s3:PutObject, s3:GetObject, s3:ListBucket, s3:DeleteObject on the S3 bucket
       - Create instance profile, add role to it
       Check with `aws iam get-role --role-name`.

    e. **Create S3 bucket** (skip if exists):
       `aws s3api create-bucket --bucket $S3_BUCKET --region $REGION --create-bucket-configuration LocationConstraint=$REGION`
       Check with `aws s3api head-bucket`.

    f. **Launch EC2 instance:**
       `aws ec2 run-instances` with all parameters (AMI, instance type, key, SG, EBS, IAM profile, Name tag).
       Wait for running state.
       Get public IP.

    g. **Save outputs:**
       - Write instance ID to `.ec2-instance-id`
       - Write public IP to `.ec2-public-ip`
       - Print connection command: `ssh -i swiss-scraper-key.pem ec2-user@<IP>`

    Error handling: `set -euo pipefail` at top. Each AWS CLI failure stops the script.

    **1. Create `scripts/deploy.sh`** — Local deploy script.

    ```bash
    #!/usr/bin/env bash
    set -euo pipefail

    # Read EC2 connection info (created by provision-ec2.sh)
    EC2_IP=$(cat .ec2-public-ip 2>/dev/null || { echo "ERROR: .ec2-public-ip not found. Run provision-ec2.sh first."; exit 1; })
    KEY_FILE="swiss-scraper-key.pem"

    if [ ! -f "$KEY_FILE" ]; then
      echo "ERROR: $KEY_FILE not found. Run provision-ec2.sh first."
      exit 1
    fi

    SSH_CMD="ssh -i $KEY_FILE -o StrictHostKeyChecking=no -o ConnectTimeout=10 ec2-user@$EC2_IP"

    echo "Deploying to $EC2_IP..."

    $SSH_CMD << 'REMOTE'
      cd ~/gen-ai-hackathon
      git pull origin master
      npm install --production
      pm2 restart ecosystem.config.cjs 2>/dev/null || pm2 start ecosystem.config.cjs
      pm2 save
      echo "Deploy complete. PM2 status:"
      pm2 list
    REMOTE

    echo "Deploy to $EC2_IP complete"
    ```

    Key features:
    - Reads .ec2-public-ip (output of provision-ec2.sh)
    - Pre-flight checks for key file and IP file
    - StrictHostKeyChecking=no for first connection
    - `pm2 restart || pm2 start` handles both first deploy and updates
    - `pm2 save` persists process list for reboot recovery
    - Prints PM2 status after deploy
