    **2. Create `src/lock.ts`** — PID-file lock with stale detection.

    ```typescript
    const LOCK_FILE = '/tmp/swiss-scraper.lock';

    export function acquireLock(): boolean
    export function releaseLock(): void
    ```

    - acquireLock: If lock file exists, read PID, check if process alive via `process.kill(pid, 0)`. If alive, return false. If dead (stale), remove lock and continue. Write current PID to lock file, return true.
    - releaseLock: Delete lock file, ignore errors.
    - Use `node:fs` synchronous APIs (existsSync, readFileSync, writeFileSync, unlinkSync).

    **3. Create `src/s3-sync.ts`** — S3 sync via AWS CLI.

    ```typescript
    export async function syncToS3(outputDir: string, s3Bucket: string, logger: Logger): Promise<'success' | 'failed'>
    ```

    - Use `execFile` from `node:child_process` (promisified) to run `aws s3 sync {outputDir} {s3Bucket}/{outputDir}`
    - Log stdout on success, log stderr as warning
    - On error (non-zero exit), log error and return 'failed' — do NOT throw (caller should not crash)
    - If s3Bucket is empty/undefined, skip sync and return 'skipped'

    **1. Create `src/scheduler.ts`** — Long-running entry point for PM2.

    ```typescript
    import cron from 'node-cron';
    import { loadConfig } from './config.js';
    import { createLogger } from './logger.js';
    import { acquireLock, releaseLock } from './lock.js';
    import { runScrape } from './scrape-pipeline.js';
    import { syncToS3 } from './s3-sync.js';
    ```

    On startup:
    - Load config, create logger
    - Log "Scheduler started" with next scheduled time info
    - Schedule cron job: `'0 2 * * *'` with `{ timezone: 'Europe/Zurich' }`

    Cron handler (async):
    1. Call `acquireLock()` — if false, log warning and return (skip this run)
    2. Try: call `runScrape('flatfox', config, logger)`
    3. If config.S3_BUCKET: call `syncToS3(meta.outputDir, config.S3_BUCKET, logger)`, update _run_meta.json with s3SyncStatus
    4. Log completion with meta
    5. Catch: log error (no retry per user decision)
    6. Finally: call `releaseLock()`

    Also add a signal handler for graceful shutdown:
    ```typescript
    process.on('SIGINT', () => { releaseLock(); process.exit(0); });
    process.on('SIGTERM', () => { releaseLock(); process.exit(0); });
    ```

    **2. Create `ecosystem.config.cjs`** at project root.

    MUST use `.cjs` extension because project has `"type": "module"` in package.json. PM2 uses `require()` to load this file.

    ```javascript
    module.exports = {
      apps: [{
        name: 'swiss-scraper',
        script: 'src/scheduler.ts',
        interpreter: 'node',
        interpreter_args: '--import tsx',
        cwd: '/home/ec2-user/gen-ai-hackathon',
        env: {
          NODE_ENV: 'production',

        log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
        error_file: '/home/ec2-user/logs/scraper-error.log',
        out_file: '/home/ec2-user/logs/scraper-out.log',
        merge_logs: true,
        max_restarts: 10,
        min_uptime: '10s',
        restart_delay: 5000,
        autorestart: true,
      }]
    };
    ```
