# Plan 41-01: PropertyMapView Component

**Phase:** 41 — Add Google Maps Street View to Property Listings
**Goal:** Install the Google Maps JS API loader package and implement the `PropertyMapView` client component with all states (loading, Street View, Map, unavailable, error).
**Depends on:** None (standalone component plan)

## Context

Key decisions from CONTEXT.md:
- **D-02:** Use Google Maps JavaScript API (not Static API / iframe embed)
- **D-03:** Default view is interactive Street View with camera heading computed toward the property
- **D-04:** Toggle view is interactive Map with a marker on the property coordinates
- **D-05:** Toggle is a UI control inside the component switching between modes
- **D-06:** Heading computed via `google.maps.geometry.spherical.computeHeading(panoramaLatLng, propertyLatLng)`
- **D-07:** If Street View unavailable (`StreetViewStatus !== OK`), auto-switch to Map, hide toggle
- **D-11:** Use lat/lng when provided; fall back to geocoding the address string via `Geocoder` if coordinates are null
- **D-13:** Required Maps JS API libraries: `maps`, `streetView`, `geometry`
- **D-14:** Component path: `web/src/components/analysis/PropertyMapView.tsx`; props: `address: string`, `latitude: number | null`, `longitude: number | null`
- **D-15:** Use `@googlemaps/js-api-loader` npm package

UI-SPEC values (exact — do not deviate):
- Map container: `h-96 w-full rounded-lg overflow-hidden border border-border`
- Toggle active: `variant="default"` (filled primary), inactive: `variant="outline"`
- Toggle size: `size="sm"`, icons: `MapPin` (Street View), `Map` (Map) from lucide-react, 4px gap before label
- Toggle hidden when Street View unavailable
- Section heading: `<h2 id="property-location-heading" className="text-xl font-semibold text-foreground">`; text: "Property Location"
- Section wrapper: `<section aria-labelledby="property-location-heading">`
- Loading state: `<Skeleton className="h-96 w-full rounded-lg" />` — no spinner, no text
- Error state: muted container `h-96` with centered `<p className="text-sm text-muted-foreground">Unable to load map</p>`
- Accessibility: `aria-pressed="true"` on active toggle; `aria-label="Property location map"` on map div; `<span className="sr-only">Interactive map showing property at {address}</span>` inside map div

Codebase conventions (CONVENTIONS.md):
- Named exports only — do NOT use `export default`
- Props interface defined above the component: `interface PropertyMapViewProps { ... }`
- `'use client'` directive required (Google Maps API is browser-only)
- `cn()` for conditional class merging
- Import `Button` from `@/components/ui/button`, `Skeleton` from `@/components/ui/skeleton`
- Icons from `lucide-react`

## Tasks

### Task 1: Install `@googlemaps/js-api-loader`
**File:** `web/package.json`
**Action:** install

Run the following from the `web/` directory:

```
cd web && npm install @googlemaps/js-api-loader
```

This adds `@googlemaps/js-api-loader` to `dependencies` in `web/package.json`. The package wraps the Google Maps script tag loading with a Promise-based API and TypeScript types included. No additional `@types/*` package is needed.

After install, verify `@googlemaps/js-api-loader` appears under `dependencies` in `web/package.json`.

### Task 2: Create `PropertyMapView` client component
**File:** `web/src/components/analysis/PropertyMapView.tsx`
**Action:** create

Create a new file at `web/src/components/analysis/PropertyMapView.tsx`. The component is a `'use client'` React component. It must be a named export.

**Full implementation specification:**

```typescript
'use client'

import { useEffect, useRef, useState } from 'react'
import { Loader } from '@googlemaps/js-api-loader'
import { MapPin, Map } from 'lucide-react'
import { Button } from '@/components/ui/button'
import { Skeleton } from '@/components/ui/skeleton'
import { cn } from '@/lib/utils'

type ViewMode = 'streetview' | 'map'

interface PropertyMapViewProps {
  address: string
  latitude: number | null
  longitude: number | null
}
```

**State variables (all inside the component function):**

```typescript
const [viewMode, setViewMode] = useState<ViewMode>('streetview')
const [loadState, setLoadState] = useState<'loading' | 'ready' | 'error'>('loading')
const [streetViewAvailable, setStreetViewAvailable] = useState(true)

const mapContainerRef = useRef<HTMLDivElement>(null)
const mapInstanceRef = useRef<google.maps.Map | null>(null)
const panoramaInstanceRef = useRef<google.maps.StreetViewPanorama | null>(null)
const markerRef = useRef<google.maps.Marker | null>(null)
```

**`useEffect` — API loading and map initialization:**

The effect runs once on mount. It must:

1. Create a `Loader` instance:
   ```typescript
   const loader = new Loader({
     apiKey: process.env.NEXT_PUBLIC_GOOGLE_MAPS_API_KEY ?? '',
     version: 'weekly',
     libraries: ['maps', 'streetView', 'geometry'],
   })
   ```

2. Call `loader.load()` inside a try/catch. On error, set `loadState('error')` and return early.

3. After load resolves, determine the target `google.maps.LatLng`:
   - If `latitude != null && longitude != null`: use `new google.maps.LatLng(latitude, longitude)`
   - Otherwise: geocode the address:
     ```typescript
     const geocoder = new google.maps.Geocoder()
     const result = await new Promise<google.maps.GeocoderResult | null>((resolve) => {
       geocoder.geocode({ address }, (results, status) => {
         resolve(status === 'OK' && results ? results[0] : null)
       })
     })
     if (!result) {
       setLoadState('error')
       return
     }
     const propertyLatLng = result.geometry.location
     ```

4. Initialize the `google.maps.Map` in `mapContainerRef.current`:
   ```typescript
   const map = new google.maps.Map(mapContainerRef.current!, {
     center: propertyLatLng,
     zoom: 16,
     disableDefaultUI: false,
   })
   mapInstanceRef.current = map
   ```

5. Add a legacy `google.maps.Marker` at `propertyLatLng` (no `mapId` required, simpler for this use case):
   ```typescript
   const marker = new google.maps.Marker({
     map,
     position: propertyLatLng,
     title: address,
   })
   markerRef.current = marker
   ```
   Add a click listener on the marker to show an `InfoWindow` with `content: address`:
   ```typescript
   const infoWindow = new google.maps.InfoWindow({ content: address })
   marker.addListener('click', () => {
     infoWindow.open({ map, anchor: marker })
   })
   ```
   Note: Do NOT use `AdvancedMarkerElement` — it requires a `mapId` on the `Map` constructor which adds unnecessary env var complexity for this phase.

6. Initialize Street View panorama using the `StreetViewService`:
   ```typescript
   const svService = new google.maps.StreetViewService()
   svService.getPanorama(
     { location: propertyLatLng, radius: 50 },
     (data, status) => {
       if (status !== google.maps.StreetViewStatus.OK || !data?.location?.latLng) {
         setStreetViewAvailable(false)
         setViewMode('map')
         setLoadState('ready')
         return
       }

       const panoramaLatLng = data.location.latLng
       const heading = google.maps.geometry.spherical.computeHeading(
         panoramaLatLng,
         propertyLatLng
       )

       const panorama = new google.maps.StreetViewPanorama(mapContainerRef.current!, {
         position: panoramaLatLng,
         pov: { heading, pitch: 0 },
         zoom: 1,
         visible: false, // hidden initially; shown when viewMode === 'streetview'
       })
       panoramaInstanceRef.current = panorama
       map.setStreetView(panorama)

       setStreetViewAvailable(true)
       setViewMode('streetview')
       setLoadState('ready')
     }
   )
   ```

7. The map container hosts both the map and panorama. Show/hide them via the panorama's `visible` property and CSS visibility of the map tiles:

   When `viewMode` changes (in a second `useEffect` watching `[viewMode, loadState]`):
   - `viewMode === 'streetview'`: call `panoramaInstanceRef.current?.setVisible(true)`
   - `viewMode === 'map'`: call `panoramaInstanceRef.current?.setVisible(false)`

   The map itself is always initialized in the same div. The panorama overlays on top when `visible: true`. This is the standard Google Maps JS API pattern for toggling panorama vs map in the same container.

**Rendered JSX structure:**

```tsx
export function PropertyMapView({ address, latitude, longitude }: PropertyMapViewProps) {
  // ... state and refs and effects ...

  return (
    <section aria-labelledby="property-location-heading">
      {/* Section heading row: title left, toggle right */}
      <div className="flex items-center justify-between mb-4">
        <h2
          id="property-location-heading"
          className="text-xl font-semibold text-foreground"
        >
          Property Location
        </h2>

        {/* Toggle — hidden when Street View unavailable */}
        {loadState === 'ready' && streetViewAvailable && (
          <div className="flex items-center gap-2">
            <Button
              size="sm"
              variant={viewMode === 'streetview' ? 'default' : 'outline'}
              aria-pressed={viewMode === 'streetview'}
              onClick={() => setViewMode('streetview')}
            >
              <MapPin className="h-3.5 w-3.5 mr-1" />
              Street View
            </Button>
            <Button
              size="sm"
              variant={viewMode === 'map' ? 'default' : 'outline'}
              aria-pressed={viewMode === 'map'}
              onClick={() => setViewMode('map')}
            >
              <Map className="h-3.5 w-3.5 mr-1" />
              Map
            </Button>
          </div>
        )}
      </div>

      {/* Loading state */}
      {loadState === 'loading' && (
        <Skeleton className="h-96 w-full rounded-lg" />
      )}

      {/* Error state */}
      {loadState === 'error' && (
        <div className="h-96 w-full rounded-lg border border-border flex items-center justify-center bg-muted">
          <p className="text-sm text-muted-foreground">Unable to load map</p>
        </div>
      )}

      {/* Map container — rendered when ready, hidden while loading/error so refs attach */}
      <div
        ref={mapContainerRef}
        aria-label="Property location map"
        className={cn(
          'h-96 w-full rounded-lg overflow-hidden border border-border',
          loadState !== 'ready' && 'hidden'
        )}
      >
        <span className="sr-only">
          Interactive map showing property at {address}
        </span>
      </div>
    </section>
  )
}
```

**Important implementation notes:**
- The `mapContainerRef` div must be in the DOM from the start (even when hidden via `hidden` class) so the Google Maps `Map` constructor can attach to it. Do NOT conditionally render it with `{loadState === 'ready' && <div ref={...}>}` — the ref would be null during initialization.
- The second `useEffect` watching `[viewMode, loadState]` must guard with `if (loadState !== 'ready') return` to avoid calling methods on null refs.
- The icon gap between icon and label text uses `mr-1` (4px) per UI-SPEC xs token.
- Toggle gap of `gap-2` (8px) matches UI-SPEC sm token between the two buttons.
- `mb-4` (16px) below the heading row matches UI-SPEC md token.
