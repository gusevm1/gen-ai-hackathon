### Task 1: Add `listing_profiles` query and render `PropertyMapView` in the analysis page
**File:** `web/src/app/(dashboard)/analysis/[listingId]/page.tsx`
**Action:** modify

Make three changes to the existing server component:

**Change 1 â€” Add import at the top** (after existing component imports):

```typescript
import { PropertyMapView } from '@/components/analysis/PropertyMapView'
import { FadeIn } from '@/components/motion/FadeIn'
```

**Change 2 â€” Add `listing_profiles` query** after the `profileName` fetch block (after line 75, before the `breakdown` destructuring):

```typescript
// Fetch listing location for map view
let listingLocation: { address: string; latitude: number | null; longitude: number | null } | null = null
const { data: listingProfile } = await supabase
  .from('listing_profiles')
  .select('address, latitude, longitude')
  .eq('listing_id', parseInt(listingId, 10))
  .maybeSingle()

if (listingProfile?.address) {
  listingLocation = {
    address: listingProfile.address,
    latitude: listingProfile.latitude ?? null,
    longitude: listingProfile.longitude ?? null,
  }
}
```

Use `.maybeSingle()` (not `.single()`) to avoid throwing when the row doesn't exist. Guard with `listingProfile?.address` â€” if the address column is null or the row is absent, `listingLocation` stays null. This implements D-10 (no error surfaced to user when row missing).

**Change 3 â€” Render `PropertyMapView` inside the return JSX**, after the closing `</div>` of the 2-column grid and before the closing `</div>` of the container:

```tsx
{/* Property location map â€” only when listing_profiles data available */}
{listingLocation && (
  <FadeIn className="mt-8">
    <PropertyMapView
      address={listingLocation.address}
      latitude={listingLocation.latitude}
      longitude={listingLocation.longitude}
    />
  </FadeIn>
)}
```

### Task 2: Add `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` to Vercel and document for local dev
**File:** `.env.local` (documentation update in project root or web/ root)
**Action:** create/modify + Vercel CLI command

**Step A â€” Add to Vercel (run this command):**


When prompted, paste the Google Maps API key value. If the key must also be set for preview/development environments, run the command again for each environment (`preview`, `development`).

The key should already be created in Google Cloud Console with the Maps JavaScript API enabled, and the HTTP referrer restricted to the Vercel deployment domain (e.g. `homematch.vercel.app/*`) per D-12 to prevent abuse.

**Step B â€” Document in `.env.local.example`** (create if not exists, or append if exists):

Check whether `web/.env.local.example` exists. If it does, append the following line. If it does not, create the file with this content:

Do NOT create or modify the actual `web/.env.local` file â€” that file is gitignored and contains real secrets. The `.env.local.example` file is committed and serves as documentation for new developers.
