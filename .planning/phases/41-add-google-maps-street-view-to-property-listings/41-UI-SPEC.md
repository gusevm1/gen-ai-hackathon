## Component Inventory

### New Components

#### `PropertyMapView` (client component)
- **Path:** `web/src/components/analysis/PropertyMapView.tsx`
- **Props:** `address: string`, `latitude: number | null`, `longitude: number | null`
- **Behavior:**
  1. Loads Google Maps JS API via `@googlemaps/js-api-loader` with libraries: `maps`, `streetView`, `geometry`
  2. Default view: Street View panorama, heading computed via `computeHeading(panoramaLatLng, propertyLatLng)`
  3. Toggle switches between Street View and Map view
  4. If Street View unavailable (`StreetViewStatus !== OK`), auto-fall to Map view, hide toggle

### Existing Components Used

| Component | From | Usage |
|-----------|------|-------|
| `Button` | `@/components/ui/button` | Toggle control (variant switching) |
| `Skeleton` | `@/components/ui/skeleton` | Loading placeholder while Maps API initializes |
| `Card` | `@/components/ui/card` | Optional container wrapper for visual consistency |

## Layout Contract

### Placement
Full-width below all score content, inside the existing `container max-w-4xl mx-auto` wrapper. Sits after the 2-column grid (breakdown + checklist), separated by `mt-8` (32px).

### Map Container
- **Height:** `h-96` (384px) on all breakpoints. Fixed height prevents layout shift.
- **Width:** Full container width (`w-full`)
- **Border radius:** `rounded-lg` (matches `--radius: 0.75rem`)
- **Overflow:** `overflow-hidden` to clip map edges to rounded corners
- **Border:** `border border-border` for visual definition against page background

### Toggle Control
- **Position:** Top-right corner of the map section, between section heading and map container
- **Layout:** Inline flex row with 8px gap between the two toggle buttons
- **Active state:** `variant="default"` (filled primary background)
- **Inactive state:** `variant="outline"` (transparent with border)
- **Button size:** `size="sm"` — compact toggle
- **Icons:** `MapPin` (lucide) for Street View, `Map` (lucide) for Map view — placed before label text with 4px gap
- **Hidden when:** Street View is unavailable (only Map shown, no toggle needed)

### Section Heading
- Text: "Property Location"
- Element: `<h2>` with `text-xl font-semibold`
- Color: `text-foreground`
- Below heading, above toggle row: no subtext

## States Contract

### Loading State
- Render `Skeleton` component at `h-96 w-full rounded-lg` while the Google Maps JS API is loading
- No spinner. The skeleton pulse animation provides sufficient loading feedback.
- Duration: typically under 2 seconds on broadband

### Street View Available (default)
- Panorama renders at full container size
- Toggle shows both "Street View" (active) and "Map" (inactive) buttons
- Panorama heading points toward the property coordinates
- Panorama pitch: 0 (horizontal)
- Panorama zoom: 1 (default FOV)

### Street View Unavailable
- Map view renders automatically
- Toggle is hidden (only one view available)
- No error message shown — map alone is valuable

### No Listing Profile (no coordinates/address)
- Entire PropertyMapView section is not rendered
- No placeholder, no error, no empty state — the section simply does not appear
- Decision source: D-10 in CONTEXT.md

### Maps API Load Failure
- Render a muted container at `h-96` with centered text: "Unable to load map"
- Text: `text-sm text-muted-foreground`
- No retry button (user can refresh the page)

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click "Street View" toggle | Switch map container to Street View panorama. Restore last panorama heading/pitch if user previously navigated. |
| Click "Map" toggle | Switch map container to interactive Map with property marker. Restore last zoom/center if user previously panned. |
| Street View drag/scroll | Pan panorama heading and pitch (native Google Maps interaction) |
| Map drag/scroll | Pan and zoom map (native Google Maps interaction) |
| Map marker click | Show default info window with property address |

## Animation Contract

| Element | Animation | Timing |
