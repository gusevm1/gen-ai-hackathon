       - 'use server' directive at top
       - `login(formData: FormData)`: call createClient() from @/lib/supabase/server, signInWithPassword with email/password from formData. On error, return { error: error.message }. On success, redirect('/dashboard').
       - `signup(formData: FormData)`: call createClient(), signUp with email/password. On error, return { error: error.message }. On success, redirect('/dashboard').
       - `logout()`: call createClient(), signOut(), redirect('/').
       - IMPORTANT: Use getUser() not getSession() for server-side auth checks (per research).

    2. Update landing page (web/src/app/page.tsx):
       - Centered login/signup form -- NO product pitch, NO branding beyond "HomeMatch" title (per user decision).
       - Client Component ('use client') for form state management.
       - Toggle between "Sign In" and "Sign Up" modes with a text link.
       - Email input + password input + submit button.
       - Display error message if login/signup fails.
       - Loading state on submit button.
       - Use shadcn/ui Button, Input, Card, Label components.
       - Call server actions via form action or fetch.
       - NOTE: Since server actions that redirect can't return errors easily, use a client-side approach: call supabase.auth.signInWithPassword directly from the browser client, then router.push('/dashboard') on success. This gives better error handling in the UI.
