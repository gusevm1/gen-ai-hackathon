3. Fix the dark mode CSS variant selector in `web/src/app/globals.css`:
   Change line 5 from:
   ```css
   @custom-variant dark (&:is(.dark *));
   ```
   To:
   ```css
   @custom-variant dark (&:where(.dark, .dark *));
   ```
   This ensures Tailwind `dark:` utilities apply to BOTH the `.dark` element itself AND its descendants. The current selector only matches descendants, which causes dark mode styles to not apply to elements directly on the `.dark` element (the html tag when using next-themes).

   ```typescript
   "use client"
   import * as React from "react"
   import { ThemeProvider as NextThemesProvider } from "next-themes"

   export function ThemeProvider({
     children,
     ...props
   }: React.ComponentProps<typeof NextThemesProvider>) {
     return <NextThemesProvider {...props}>{children}</NextThemesProvider>
   }
   ```

   - Add `suppressHydrationWarning` to the `<html>` tag (required by next-themes to prevent hydration mismatch)

     - `defaultTheme="system"` (respects OS preference on first visit)
     - `enableSystem` (watches for OS theme changes)
     - `disableTransitionOnChange` (prevents flash of unstyled content during theme switch)
   - Update the metadata title to "HomeMatch" and description to "AI-powered property matching"
   - Keep fonts as-is (Geist, Geist_Mono)

4. Create `web/src/app/(dashboard)/layout.tsx` as a minimal skeleton:
   ```typescript
   import { SidebarProvider, SidebarInset } from "@/components/ui/sidebar"

   export default function DashboardLayout({
     children,
   }: {
     children: React.ReactNode
   }) {
     return (
       <SidebarProvider defaultOpen={true}>
         {/* AppSidebar will be added in Plan 02 */}
         <SidebarInset>
           <main className="flex-1 p-4">
             {children}
           </main>
         </SidebarInset>
       </SidebarProvider>
   ```
