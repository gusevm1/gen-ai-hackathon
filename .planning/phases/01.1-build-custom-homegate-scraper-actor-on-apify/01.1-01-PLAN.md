Purpose: Replace the paid third-party actor ($20/month) with a custom actor that uses only platform credits. This is the core scraping engine that the HomegateAdapter in the main project will call.

Output: A complete, deployable Apify actor project in `actors/homegate-scraper/` that extracts listings from Homegate search pages via `window.__INITIAL_STATE__` JSON, paginates automatically, and pushes results to an Apify Dataset.

  <action>
    **src/parsers/initial-state.ts:**
    - Export `parseInitialState($: CheerioAPI): Record<string, unknown> | null`
    - Find `<script>` tag containing `window.__INITIAL_STATE__`
    - Extract JSON using `split('=').slice(1).join('=').trim()` pattern (handles `=` inside JSON)
    - Wrap in try/catch, return null on parse failure with console warning
    - Do NOT use CSS class selectors (anti-pattern from research)

    **src/routes.ts:**
    - Export the search page request handler function (not a Crawlee router -- use inline handler)
    - Import parseInitialState
    - Extract `resultList.search.fullSearch.result` from parsed state
    - Get `listings[]` and `pageCount` from result
    - For each listing, call a `transformListing()` function that maps HomegateListingCard fields to a flat object compatible with normalizeHomegate() expectations:
      - `url`: Build from localization.de.urls.detail or construct from id: `https://www.homegate.ch/{rent|buy}/{id}`
      - `title`: Extract from title.de (or first available locale)
      - `price`: Extract from prices.rent.gross (for rent) or prices.buy.price (for buy)
      - `rooms`: characteristics.numberOfRooms
      - `livingSpace`: characteristics.livingSpace
      - `floor`: characteristics.floor
      - `yearBuilt`: characteristics.yearBuilt
      - `address`: Build structured object from address fields (street, postalCode as zip, locality as city)
      - `latitude`/`longitude`: From address.geoCoordinates
      - `description`: description.de (or first locale)
      - `images`: Map pictures[].url
      - `propertyType`: Derive from categories or offerType
      - `agencyName`: lister.name
      - `listingType`: From listingType.type (RENT -> rent, BUY -> buy)
      - `currency`: From prices.currency, default 'CHF'
    - Push transformed listings via `Actor.pushData()`
    - On first page (no userData.page or page === 1): enqueue remaining pages using `?ep=` parameter up to pageCount (or limited by maxItems / 20)
    - Log progress: page number, listings found, total pages

    **src/main.ts:**
    - Import Actor from apify, CheerioCrawler and log from crawlee
    - `await Actor.init()`
    - Get input via `Actor.getInput<ActorInput>()`
    - Create proxy configuration: `Actor.createProxyConfiguration()` (no specific group -- let Apify decide, keeps credits low)
    - Create CheerioCrawler with:
      - proxyConfiguration
      - maxConcurrency: 5 (conservative per research)
      - maxRequestRetries: 3
      - requestHandlerTimeoutSecs: 60
      - The request handler from routes.ts (pass crawler reference for addRequests)
    - Build start URLs based on listingType:
      - rent: `https://www.homegate.ch/rent/real-estate/switzerland/matching-list`
      - buy: `https://www.homegate.ch/buy/real-estate/switzerland/matching-list`
      - both: both URLs
    - `await crawler.run(startUrls)`
    - `await Actor.exit()`
    - Handle errors gracefully: wrap in try/catch, log errors, still call Actor.exit()
