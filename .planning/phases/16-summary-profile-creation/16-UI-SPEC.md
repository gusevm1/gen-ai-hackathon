## Interaction Contracts

### Summary Card Appearance
- Trigger: backend returns `ready_to_summarize: true` with `extracted_preferences` object
- `ConversationPhase` type gains `'summarizing'` state
- Card renders inside the chat thread (below last assistant message), not as overlay
- Entry animation: fade-in with subtle slide-up (opacity 0 to 1, translateY 8px to 0, 300ms ease-out) -- Claude's discretion area
- Chat input is disabled (greyed out) while summary card is visible -- Claude's discretion area, recommended for focus

### Click-to-Edit Fields
- Default state: field value displayed as static text, pencil icon (Pencil from lucide-react, 14px) visible on row hover
- Edit state: clicking value or pencil icon replaces static text with an `<Input>` component, auto-focused
- Save: on blur, the edited value commits to local React state; no API call until Confirm
- Cancel: pressing Escape reverts to previous value
- For null/empty essential fields: render as "Click to add {field name}" in `text-muted-foreground` italic

### Importance Dropdowns
- Five `<Select>` components, one per category: location, price, size, features, condition
- Options: critical, high, medium, low
- Default: pre-filled from AI extraction; user can change before confirming

### Dealbreaker Checkboxes
- Three `<Checkbox>` components next to: budget, rooms, living space
- Label: "Hard limit"
- Default: pre-filled from AI extraction (maps from "dealbreaker" language cues)

### Confirm Flow
- Button: full-width at card footer, `variant="default"` (primary color)
- Loading: button disabled, spinner (Loader2 from lucide-react, animate-spin) + "Creating profile..." text
- Success: `router.push(/profiles/${newProfileId})` -- navigates to new profile detail page
- Error: red text below button inside card footer, e.g., "Failed to create profile. Please try again."

### "Back to Chat" Escape
- Small text link below the confirm button: "Continue chatting instead" -- Claude's discretion area, recommended
- Clicking it sets phase back to `'chatting'`, removes summary card, re-enables chat input
- Previous messages and extracted preferences remain in state for potential re-extraction

## State Machine

```
idle --> naming --> chatting --> summarizing --> (profile created, redirect)
                                    |
                                    v
                                chatting (via "Continue chatting instead")
```

States relevant to this phase:
- `chatting` (existing): user is in conversation, backend may return `ready_to_summarize`
- `summarizing` (new): summary card visible, chat input disabled, user can edit fields and confirm
- On confirm success: redirect to `/profiles/${id}`, component unmounts
- On "Continue chatting instead": revert to `chatting`, re-enable input
