**Pipeline steps inside `score_listing()` endpoint:**

0. **Cache check** -- same as current (call `get_analysis`, return cached if found). Note: DB-02 schema_version filtering is handled by Plan 03 updating `get_analysis()`. For now, the existing get_analysis works fine.

1. **Parse preferences** from request.

2. **ListingProfile lookup** -- `await asyncio.to_thread(get_listing_profile, request.listing_id)`.

3. **If ListingProfile found (happy path):**
   a. `listing = adapt_profile_to_listing(profile)` -- convert to FlatfoxListing
   b. `proximity_data = adapt_profile_amenities(profile)` -- convert amenities
   c. **Score deterministic criteria:**
      - Call `synthesize_builtin_results(preferences, listing)` for built-in fields (budget/rooms/living_space)
      - Loop over `preferences.dynamic_fields`, route by `field.criterion_type`:
        - `CriterionType.PRICE` -> `score_price(field, listing)`
        - `CriterionType.DISTANCE` -> need `actual_km` from proximity_data. Extract amenity category from field name using keyword matching. Look up in proximity_data to find nearest distance. Call `score_distance(field, listing, actual_km)`.
        - `CriterionType.SIZE` -> `score_size(field, listing)`
        - `CriterionType.BINARY_FEATURE` -> `score_binary_feature(field, listing)`
        - `CriterionType.PROXIMITY_QUALITY` -> `score_proximity_quality(field, listing, proximity_data)`
        - `CriterionType.SUBJECTIVE` or `None` -> skip (handled by subjective scorer)
      - For each scored field, create `FulfillmentResult(criterion_name=field.name, fulfillment=result, importance=field.importance)`
   d. **Score subjective criteria** -- call `await claude_scorer.score_listing(listing, preferences, nearby_places=proximity_data or None)` which returns `(subjective_results, summary_bullets)`. The subjective scorer internally filters to only subjective fields.
   e. **Merge results** -- combine deterministic FulfillmentResults + subjective FulfillmentResults into one list.
   f. **Aggregate** -- `overall_score, match_tier = compute_weighted_score(all_results)`
   g. **Build response** -- `ScoreResponse(overall_score=overall_score, match_tier=match_tier, summary_bullets=summary_bullets, criteria_results=[to_criterion_result(r) for r in all_results], schema_version=2, enrichment_status="available", categories=[], checklist=[], language=preferences.language)`

4. **If ListingProfile NOT found (degradation path):**
   a. If `ALLOW_CLAUDE_FALLBACK` is True: preserve the OLD pipeline (fetch from Flatfox API, geocode, fetch proximity, scrape page data, call claude_scorer, build temporary response). Keep this code from the current router but wrap it in this branch. Import the necessary services: `flatfox_client`, `geocode_listing`, `fetch_all_proximity_data`.
   b. If `ALLOW_CLAUDE_FALLBACK` is False (INT-04): return immediately:
      ```python
      return ScoreResponse(
          overall_score=0,
          match_tier="poor",
          summary_bullets=[
              "Scoring is not yet available for this listing's area.",
              "Pre-computed listing data has not been generated for this property.",
              "Check back later as coverage expands.",
          ],
          criteria_results=[],
          schema_version=2,
          enrichment_status="unavailable",
          categories=[],
          checklist=[],
          language=preferences.language,
      )
      ```
      Do NOT save this "unavailable" response to the analyses table -- just return it directly.

5. **Save analysis** (only for successful scores, not "unavailable"):
   - Build `score_data = result.model_dump()` (using the ScoreResponse)
   - Add metadata: `listing_title`, `listing_address`, `listing_rooms`, `listing_object_type` from the adapted listing
   - Also populate `fulfillment_data` key with the criteria_results for the DB column (migration 007)
   - Call `supabase_service.save_analysis(...)` via asyncio.to_thread, fire-and-forget with try/except

**Distance criteria resolution for adapted profiles:**
For `CriterionType.DISTANCE` fields, the `actual_km` comes from pre-computed proximity data. Create a helper function `_resolve_distance_km(field_name: str, proximity_data: dict[str, list[dict]]) -> Optional[float]` that:
- Normalizes the field name to lowercase
- Tries common keywords to match amenity categories: extract words from the field name, match against proximity_data keys
- If found, returns the first entry's distance_km
- If not found, returns None (score_distance will return None, and the criterion is skipped per HA-02)

**Error handling:** Wrap the deterministic + subjective scoring in try/except. If scoring fails, raise HTTPException(502). Log errors with listing ID for debugging.
