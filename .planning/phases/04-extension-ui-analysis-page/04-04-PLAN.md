<objective>
Add listing image analysis to the Claude scoring pipeline -- enables evaluation of property condition, view quality, interior finishes, and natural light from listing photos.

Purpose: Text-only scoring misses visual aspects that are critical to property evaluation. Flatfox listings include property photos that show condition, views, kitchen/bathroom quality, and natural light. By passing these images to Claude alongside the text prompt, the scoring becomes significantly more comprehensive. This is a separate plan (per user decision) to avoid blocking the core extension UI work.

Output: Backend scoring pipeline that includes listing images in Claude prompts when available, with graceful fallback to text-only when images cannot be fetched.
</objective>

    - Test 4: URLs are deduplicated and limited to max 5 images (token cost control)

  <action>
1. Add `get_listing_image_urls` method to `FlatfoxClient` in `backend/app/services/flatfox.py`:
   - Takes `slug: str` and `pk: int` parameters
   - Fetches the listing detail HTML page: `GET https://flatfox.ch/en/flat/{slug}/{pk}/`
   - Parses the HTML to extract image URLs:
     - First, try `<meta property="og:image" content="...">` for the main listing image
     - Then, find all `<img>` tags with `src` or `srcset` containing `/thumb/` or `/media/`
     - Extract full URLs from srcset (take the highest resolution variant)
   - Deduplicate URLs (same image may appear in og:image and srcset)
   - Limit to max 5 images (Claude token cost: ~1334 tokens per 1000x1000 image, 5 images = ~6700 tokens)
   - Wrap in try/except: return empty list on any error (HTTP error, parse error, timeout)
   - Use the existing `get_client()` httpx client (but need to create a separate request without the base_url since this fetches HTML, not JSON API)
   - Actually, create a new httpx request directly since the base_url is set to the API path. Use `httpx.AsyncClient()` locally or use `self._client` with an absolute URL.

2. Update `backend/app/models/listing.py`:
   - No model changes needed (image_urls are fetched separately from the listing detail page, not from the API response)

3. Create `backend/tests/test_image_scoring.py`:
   - Test `get_listing_image_urls` with mocked HTML responses containing various image patterns
   - Test graceful fallback on HTTP errors
   - Test deduplication and 5-image limit
