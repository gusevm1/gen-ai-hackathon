2. Create `extension/src/entrypoints/content/index.tsx`:
   - `defineContentScript` with:
     - `matches: ['*://*.flatfox.ch/*']`
     - `cssInjectionMode: 'ui'`
   - In `main(ctx)`:
     - Create a Shadow DOM overlay for the FAB using `createShadowRootUi(ctx, { name: 'homematch-fab', position: 'overlay', anchor: 'body', ... })`
     - In `onMount`: create a div, append to container, create React root, render `<App />`
     - In `onRemove`: unmount React root
     - Call `fab.mount()` to inject

3. Create `extension/src/entrypoints/content/App.tsx`:
   - Root React component managing all content script state
   - State: `scores: Map<number, ScoreResponse>`, `loadingIds: Set<number>`, `isScoring: boolean`, `error: string | null`
   - `handleScore` function:
     a. Get session via `browser.runtime.sendMessage({ action: 'getSession' })`
     b. If no session/JWT: set error "Please log in via the HomeMatch popup first"
     c. Extract listing PKs via `extractVisibleListingPKs()` from `@/lib/flatfox`
     d. If no PKs found: set error "No listings found on this page"
     e. Set all PKs as loading, set `isScoring: true`
     f. Call `scoreListings(pks, jwt, onResult)` from `@/lib/api` where `onResult` callback removes PK from loading set and adds to scores map
     g. On completion: set `isScoring: false`
     h. On error: set error message, clear loading
   - Render `<Fab>` component with `onClick={handleScore}`, `isScoring` prop, `scoredCount` prop
   - **Badge injection strategy:** After each score arrives (via onResult callback), use `findListingCardElement(pk)` to locate the listing card, then inject a badge into the page DOM. Create a container div, use `createRoot` to render `<ScoreBadge>` into it, and insert it relative to the listing card. The badge does NOT need Shadow DOM -- it's simple enough that inline styles suffice (avoids the complexity of per-badge shadow roots).
   - Actually, for style isolation, render badges using inline styles only (no Tailwind classes) since they're injected into the host page DOM, not inside the shadow root. This avoids needing per-badge Shadow DOM while keeping styles isolated from Flatfox CSS.

4. Create `extension/src/entrypoints/content/components/Fab.tsx`:
   - Props: `{ onClick: () => void; isScoring: boolean; scoredCount: number; error: string | null }`
   - Fixed position button: bottom-right corner (bottom: 24px, right: 24px)
   - Circular button, 56px diameter, emerald-500 background, white icon
   - Icon: Use a simple SVG sparkle/star icon (inline SVG to avoid lucide-react dependency issues in Shadow DOM)
   - When `isScoring`: show a spinner animation, disable button
   - When `scoredCount > 0`: show small badge count on the FAB
   - If `error`: show error tooltip/message below FAB that auto-dismisses after 5 seconds
   - Use Tailwind classes (we're inside the Shadow DOM overlay, Tailwind is injected via style.css)

5. Create `extension/src/entrypoints/content/components/LoadingSkeleton.tsx`:
   - Simple pulsing placeholder shown in badge position while score is loading
   - Small rectangle (80px x 32px) with gray background and animate-pulse
   - Inline styles (injected into host page, not shadow DOM)
