<objective>
Content script UI for Flatfox.ch -- the core user-facing extension experience with FAB, score badges in Shadow DOM, and summary panels.

Purpose: This is the primary value delivery of the entire product. When a user browses Flatfox search results and clicks the floating action button, the extension extracts listing IDs from the page, scores them via the edge function, and progressively renders color-coded score badges next to each listing. Each badge is injected using `createShadowRootUi` with `position: 'inline'` for style isolation from Flatfox CSS (per locked decision). Clicking a badge reveals a 3-5 bullet summary with a link to the full analysis on the website.

Output: Working content script that activates on flatfox.ch, shows FAB, scores listings on demand, renders Shadow DOM badges with expandable summary panels.
</objective>

**IMPORTANT:** Delete the old `extension/src/entrypoints/content.ts` file first. WXT uses the directory-based entry (`content/index.tsx`) instead.

**CRITICAL CONSTRAINT (locked decision):** All badges and panels MUST be injected using Shadow DOM (`createShadowRootUi` with `position: 'inline'`), NOT inline styles in the host page DOM. This provides style isolation from Flatfox CSS. The main app FAB already uses `createShadowRootUi` -- apply the same pattern for per-badge injection.

1. Create `extension/src/entrypoints/content/style.css`:
   - Import Tailwind: `@tailwind base; @tailwind components; @tailwind utilities;`
   - Set `font-size: 16px` on `:host` selector (fixes rem unit issue in Shadow DOM -- rem resolves to host page's root font-size, not shadow root)
   - Basic reset for the shadow root container

2. Create `extension/src/entrypoints/content/index.tsx`:
   - `defineContentScript` with:
     - `matches: ['*://*.flatfox.ch/*']`
     - `cssInjectionMode: 'ui'`
   - In `main(ctx)`:
     - Create a Shadow DOM overlay for the FAB using `createShadowRootUi(ctx, { name: 'homematch-fab', position: 'overlay', anchor: 'body', ... })`
     - In `onMount`: create a div, append to container, create React root, render `<App ctx={ctx} />`
     - In `onRemove`: unmount React root
     - Call `fab.mount()` to inject
   - IMPORTANT: Pass `ctx` (ContentScriptContext) to `<App>` so it can create per-badge shadow roots

3. Create `extension/src/entrypoints/content/App.tsx`:
   - Root React component managing all content script state
   - Props: `{ ctx: ContentScriptContext }` -- needed for `createShadowRootUi` calls for per-badge injection
   - State: `scores: Map<number, ScoreResponse>`, `loadingIds: Set<number>`, `isScoring: boolean`, `error: string | null`, `openPanelId: number | null`
   - `handleScore` function:
     a. Get session via `browser.runtime.sendMessage({ action: 'getSession' })`
     b. If no session/JWT: set error "Please log in via the HomeMatch popup first"
     c. Extract listing PKs via `extractVisibleListingPKs()` from `@/lib/flatfox`
     d. If no PKs found: set error "No listings found on this page"
     e. Set all PKs as loading, set `isScoring: true`
     f. For each loading PK, inject a loading skeleton badge using `createShadowRootUi(ctx, { name: 'homematch-badge-{pk}', position: 'inline', anchor: findListingCardElement(pk), append: 'after', ... })` -- this creates a Shadow DOM loading placeholder next to each listing card
     g. Call `scoreListings(pks, jwt, onResult)` from `@/lib/api` where `onResult` callback:
        - Finds the existing shadow root for that PK (by name attribute)
        - Updates its content to show the real `<ScoreBadge>` + `<SummaryPanel>` instead of loading skeleton
        - OR: unmount the loading skeleton shadow root and create a new one with the badge
     h. On completion: set `isScoring: false`
     i. On error: set error message, clean up loading skeletons
   - **Per-badge Shadow DOM injection strategy:**
     Each badge gets its own `createShadowRootUi` call with `position: 'inline'` anchored to the listing card element found by `findListingCardElement(pk)`. This gives every badge full style isolation from Flatfox CSS. The `cssInjectionMode: 'ui'` ensures Tailwind CSS is bundled into each shadow root automatically.
   - `handleTogglePanel(id: number)`: toggle `openPanelId` (set to id if different, null if same). Since each badge is in its own React root inside a shadow DOM, use a custom DOM event `homematch:panel-toggle` dispatched on `document` to broadcast the open panel ID. Each badge root listens for this event to update its panel open state.
   - Render `<Fab>` component with `onClick={handleScore}`, `isScoring` prop, `scoredCount` prop
