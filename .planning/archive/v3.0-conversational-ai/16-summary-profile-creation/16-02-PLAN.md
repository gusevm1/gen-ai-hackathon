**Component structure:**

1. **State initialization:** Call `mapExtractedPreferences(extractedPreferences)` on mount (useMemo) to get initial `Preferences` object. Store in `useState<Preferences>`.

2. **Editing state:** Use `useState<string | null>(null)` for `editingField` -- tracks which field is currently being edited (field key string or null).

3. **Card layout with 4 sections per UI-SPEC.md:**

   **Section 1 "Location & Type":** location (Input click-to-edit), offerType (Select: RENT/SALE), objectCategory (Select: APARTMENT/HOUSE/ANY), language (Select: de/en/fr/it)

   **Section 2 "Budget & Size":** budgetMin/budgetMax (Input number click-to-edit), budgetDealbreaker (Checkbox "Hard limit"), roomsMin/roomsMax (Input number), roomsDealbreaker (Checkbox), livingSpaceMin/livingSpaceMax (Input number), livingSpaceDealbreaker (Checkbox), floorPreference (Select: any/ground/not_ground), availability (Input text)

   **Section 3 "Preferences & Amenities":** features (Badge list with X to remove + Input to add, placeholder "Add a feature..."), softCriteria (Badge list with X to remove + Input to add, placeholder "Add a preference...")

   **Section 4 "Importance Levels":** 5 Select dropdowns for location/price/size/features/condition importance, options: critical/high/medium/low

4. **Click-to-edit pattern:**
   - Default: field value as static `<span>`, pencil icon (Pencil from lucide-react, size 14) visible on row hover via `group-hover:opacity-100 opacity-0` on the icon
   - For null/empty text fields: show "Click to add {field name}" in `text-muted-foreground italic`
   - For null numbers: show "Not set" in `text-muted-foreground`
   - Clicking value or pencil: set `editingField` to field key, render `<Input>` auto-focused
   - On blur: commit value to preferences state, set `editingField` to null
   - On Escape keydown: revert to previous value, set `editingField` to null

5. **Badge list pattern (features, softCriteria):**
   - Map over array items rendering `<Badge variant="secondary">` with text and X icon button to remove
   - Below badges: `<Input>` with placeholder, on Enter keydown add value to array and clear input

6. **Confirm flow:**
   - `<Button>` full width in CardFooter, text "Confirm & Create Profile", `variant="default"` (primary color)
   - On click: set `isCreating` state true, call `createProfileWithPreferences(profileName, preferences)` (import from `@/app/(dashboard)/profiles/actions`)
   - Success: call `onProfileCreated(profileId)`
   - Error: set `error` state string, display as `<p className="text-sm text-destructive">` below button
   - Loading: button disabled, show `<Loader2 className="mr-2 h-4 w-4 animate-spin" />` + "Creating profile..."

7. **"Continue chatting instead" link:**
   - Below the confirm button in CardFooter: `<button className="text-sm text-muted-foreground hover:text-foreground underline">Continue chatting instead</button>`
   - On click: call `onContinueChatting()`

8. **Card wrapper:**
   - Use `Card` with className `rounded-xl border bg-card` (matches UI-SPEC dominant/secondary color split)
   - CardHeader: title "Your Preference Summary" (text-xl font-semibold), subtitle "Profile: {profileName}" (text-sm text-muted-foreground)
   - Entry animation: `animate-in fade-in slide-in-from-bottom-2 duration-300` (Tailwind animate utilities)

9. **Styling tokens from UI-SPEC:**
   - Card padding: `p-4` (16px sides) `py-6` (24px top/bottom)
   - Section gap: `space-y-8` (32px between sections)
   - Field gap within section: `space-y-4` (16px)
   - Section headings: `text-lg font-semibold` (18px/600)
   - Field labels: `text-xs` (12px)
   - Field values: `text-sm` (14px)
